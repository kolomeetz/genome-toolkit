# UX Audit: Main Sections — 2026-04-12

Dual audit by Claude Opus 4.6 + Codex (gpt-5-codex, high reasoning).

## Methodology
Two independent audits — component analysis + memory context (Claude) and code-level gap analysis (Codex) — converged on key issues.

---

## 1. SNP Browser

### User Questions
1. Which variants are most clinically important?
2. How rare/common is each variant in my ancestry?
3. Which variants map to my current symptoms?
4. Are any tied to medications I'm taking?
5. How confident are these calls (genotyped vs imputed, ClinVar review)?
6. Do any variants interact within the same pathway?
7. What lifestyle/supplement actions are associated?
8. How do my variants compare with published odds ratios?
9. How can I export a focused one-pager for my clinician?

### Gaps
- Table shows only locus metadata — no effect size, allele frequency, review level
- Population frequency hidden in VariantDrawer — requires click per variant
- No narrative/grouping — raw table only, no pathway summaries
- Goal pills hardcoded — can't create custom filters

### Proposals
1. Add effect size badge, frequency sparkline, ClinVar review level columns
2. Inline population frequency bar in table rows
3. Pathway summary panel grouping variants by system
4. User-defined filter presets replacing static goal pills
5. Auto-generated "Top 10 for your doctor" list with export

## 2. Mental Health Dashboard

### User Questions
1. Which pathways explain my current symptoms?
2. What should I do first — labs, supplements, therapy?
3. Which genes interact (COMT + MAO-A)?
4. Which variants map to anxiety vs depression vs ADHD?
5. What should I share with my psychiatrist?
6. How strong is the evidence?
7. How does my profile compare to population?

### Gaps
- Category filters non-functional: `gene.categories` always `[]`
- Only 5 hardcoded systems — missing inflammation, circadian, gut-brain, hormones
- GENE_META populated for only 3 genes
- No action prioritization or roadmap

### Proposals
1. Populate categories from vault tags → working symptom filters
2. Dynamic systems from config/vault metadata
3. Action Roadmap with severity/evidence ranking
4. Auto-generate narrative from actions + vault data
5. Symptom-first navigation ("worried about anxiety? here's what your genome says")

---

## 3. PGx / Drugs

### User Questions
1. Any contraindications with my current prescriptions?
2. Should my antidepressant dose be adjusted?
3. How do I metabolize pain meds / stimulants / anesthetics?
4. Which enzyme explains my side effects?
5. What note should I hand my prescriber?
6. How do recreational substances interact with my enzymes?
7. Any multi-enzyme interactions?

### Gaps
- Fixed filter chips — no "my medications" input
- usePGxData maps enzymes independently — no cross-enzyme interactions
- DrugCard: descriptive prose only — no dosage table, evidence grade
- Substances section disconnected from metabolizer status

### Proposals
1. "My Medications" input pinning drugs to top
2. Multi-enzyme aggregation per drug with CPIC/DPWG levels
3. Structured dosing guidance with predicted plasma delta
4. Substance ↔ enzyme linking with risk scores
5. Interaction checker for dangerous combos

---

## 4. Addiction & Reward

### User Questions
1. Why do I react differently to substances?
2. Which genes increase reward sensitivity / impulsivity?
3. Which substances to avoid or approach carefully?
4. Where do addiction genes overlap with mental health?
5. How does this affect medication-assisted treatment?
6. What should my counselor know?

### Gaps
- Filters only by evidence tier — no substance/behavior/recovery stage filter
- Only 5 static systems — missing nicotine, stimulants, behavioral addictions
- Substance cards static, no personalization
- Gene detail generic, no addiction-specific narratives

### Proposals
1. Expand systems from vault tags
2. Substance-first navigation
3. Addiction-specific gene narratives (coping strategies, therapy pointers)
4. Cross-section links (Mental Health, PGx overlap)
5. Recovery stage filters

## 5. Risk Landscape

### User Questions
1. How do my genes affect risk for leading causes of death?
2. What to monitor annually vs quarterly?
3. Can I input my age, sex, ancestry?
4. Absolute vs population risk?
5. What to show my doctor?
6. What specific actions reduce each risk?

### Gaps
- Demographics hardcoded (males, 30–44, European)
- Risk bars are heuristic, not PRS
- Static cause list — no custom priorities
- No timeline/severity ranking in expanded details

### Proposals
1. Demographic inputs (age, sex, ancestry)
2. Honest risk communication with confidence levels
3. Action Timeline with screening/lab schedules
4. Custom priority pinning
5. Lab/lifestyle integration (future)
